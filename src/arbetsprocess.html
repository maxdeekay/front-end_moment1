Hur ser min arbetsprocess ut när det kommer till utveckling och publicering?

För att automatisera så mycket som möjligt när det kommer till utveckling och publicering använder jag mig av Parcel, GitHub och Netlify.

Min arbetsprocess ser ungefär ut som följande:

  1. En förändring i koden görs.
  2. Förändringen förbereds i Git, jag lägger till ett commit-meddelande och pushar till mitt remote-repo, oftast i en separat branch.
  3. När en förändring är redo att gå live så använder jag "git merge" för att lägga till mina förändringar till main branch.
  4. När Netlify känner av en förändring i min main branch så kör den själv mitt build-kommando "npm run build" och kompilerar HTML, CSS samt JS och speglar dem på webbplatsen.

Det som är automatiserat med Parcel är optimering av bilder, bakåtkompabilitet, kopiering av arbetsfiler till mapp för publicering med optimerade filer samt minifiering av HTML, CSS och JS.

Det som är automatiserat med GitHub och Netlify är publicering av webbplatsen.
